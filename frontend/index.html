Setup Guide

How to go live with your WhatsApp AI assistant.

1️⃣ Configure Environment Variables

Set these in your application.properties or as environment variables before starting the server: 

# AI (choose one)
ANTHROPIC_API_KEY=sk-ant-...
AI_PROVIDER=anthropic

# OR
OPENAI_API_KEY=sk-...
AI_PROVIDER=openai

# Twilio WhatsApp
TWILIO_ACCOUNT_SID=ACxxxxxxxx
TWILIO_AUTH_TOKEN=xxxxxxxx
TWILIO_WHATSAPP_NUMBER=whatsapp:+14155238886

# Admin login
ADMIN_EMAIL=admin@yourbusiness.com
ADMIN_PASSWORD=your-secure-password

# JWT
JWT_SECRET=a-very-long-random-secret-string

2️⃣ Build & Run the Spring Boot Backend

cd backend
mvn clean package -DskipTests
java -jar target/assistant-1.0.0.jar

# Or with Maven wrapper:
./mvnw spring-boot:run

The API runs on http://localhost:8080. The H2 database console is at /h2-console.

3️⃣ Open the Admin Dashboard

Open frontend/index.html in your browser (or serve it from any static host). The dashboard talks to the backend at http://localhost:8080 by default. Change API_BASE at the top of the <script> tag to match your deployed server.

4️⃣ Configure the Twilio Webhook

In your Twilio Console, go to Messaging → Senders → WhatsApp Sandbox and set: 

Webhook URL (When a message comes in):
https://YOUR-DOMAIN.com/api/webhook/whatsapp

Method: HTTP POST

Use ngrok for local testing: ngrok http 8080 → copy the HTTPS URL.

5️⃣ Fill in Your Business Profile

Go to Business Profile → fill all fields → Save. Then add your Services, FAQs, and Policies. Use Preview Bot to test the AI before going live.

🗄 Database

Default: H2 file-based — zero setup, data persists in ./data/whatsappai.mv.db.
For production: switch to MySQL by uncommenting the MySQL block in application.properties.
